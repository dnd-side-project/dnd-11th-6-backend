=== 커서 기반 페이지네이션 설명

==== Query Parameter
include::{snippets}/snap-controller-test/find-snaps-in-meeting/query-parameters.adoc[]

`*cursorId*`

1. 처음 조회 시:

- 처음 데이터를 요청할 때는 cursorId가 없으므로, cursorId 값을 0 또는 null로 설정합니다.
- 이 경우 서버는 첫 번째 페이지의 데이터를 반환하고, 이와 함께 다음 페이지를 요청할 때 사용할 nextCursorId를 응답으로 보내줍니다.

2. 다음 페이지 조회 시:

- 첫 번째 응답에서 받은 nextCursorId를 다음 요청의 cursorId에 넣어서 보냅니다.
- 이를 통해 서버는 해당 커서 이후의 데이터를 반환하게 됩니다.
- 예를 들어, 첫 번째 요청에서 받은 nextCursorId가 1234라면, 다음 요청 시 cursorId에 1234를 넣어서 요청하면 됩니다. 이렇게 하면 서버는 1234 이후의 데이터를 반환하게 됩니다.

`*limit*`

- **`limit`**은 한 번의 요청에서 조회할 데이터의 최대 개수를 설정하는 값입니다.
- 예를 들어, `limit` 값을 `20`으로 설정하면, 서버는 한 번의 요청에서 최대 20개의 데이터를 반환합니다.
- 만약 **`limit`**을 설정하지 않으면, 기본값으로 **10**이 적용됩니다. 즉, 별도로 `limit` 값을 지정하지 않으면 서버는 기본적으로 10개의 데이터를 반환합니다.

===== Response Body
[source,json,options="nowrap"]
----
{
  "status" : 200,
  "data" : {
    "nextCursorId" : 6,
    "data" : [],
    "count" : 214,
    "hasNext" : true
  }
}
----

1. **`data.nextCursorId`:**
- 이 필드는 다음 페이지를 조회할 때 사용할 커서 ID입니다.
- 다음 페이지 데이터를 요청하려면 이 값을 ``cursorId``로 그대로 사용하면 됩니다.
- 예를 들어, 첫 번째 응답에서 ``data.nextCursorId``가 ``5678``이라면, 다음 요청 시 이 값을 ``cursorId``로 설정하여 서버에 요청하면 됩니다.

2. **`data.data`:**
- 이 필드는 실제로 조회된 데이터가 들어 있는 영역입니다.
- 예를 들어, 이곳에 상품 목록, 게시글 목록 등 실제로 클라이언트가 필요한 데이터들이 들어 있습니다.
- limit로 요청한 개수만큼 데이터가 반환됩니다.

3. **`count`:**
- 전체 데이터의 개수를 나타내는 필드입니다.
- 이를 통해 전체 데이터가 얼마나 되는지 파악할 수 있습니다.
- data.data필드의 개수가 아닌 데이터의 전체 개수입니다.

4. **`hasNext`:**
- 다음 페이지가 있는지를 나타내는 불리언 값입니다.
- ``true``일 경우 다음 페이지가 있다는 의미이며, 이때 ``data.nextCursorId``를 사용하여 다음 페이지 데이터를 요청할 수 있습니다.
- ``false``일 경우 더 이상 데이터가 없다는 의미이므로, 추가 요청을 하지 않아도 됩니다.
- 따라서, 이 값이 ``true``일 때만 ``nextCursorId``를 활용하여 추가 데이터를 요청하면 됩니다.

*`요약`*

- 응답에서 **``data.data``**에 실제 데이터가 들어 있습니다.
- 전체 데이터의 개수는 **``count``**로 확인할 수 있습니다.
- **``hasNext``**가 ``true``일 때, **``data.nextCursorId``**를 **``cursorId``**로 설정해 다음 페이지 데이터를 요청합니다.
- **``hasNext``**가 ``false``면 더 이상 추가 데이터를 요청할 필요가 없습니다.
